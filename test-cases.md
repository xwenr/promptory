## 一、认证模块

### TC-AUTH-01：邮箱注册

| 项目 | 内容 |
|------|------|
| **前置条件** | Supabase 已配置（.env.local 有 URL 和 ANON_KEY） |
| **步骤** | 1. 访问 `/auth/login`<br>2. 切换到注册模式<br>3. 输入邮箱和密码<br>4. 点击注册 |
| **预期结果** | 注册成功提示，确认邮件发送（或直接登录，取决于 Supabase 配置） |

### TC-AUTH-02：邮箱登录

| 项目 | 内容 |
|------|------|
| **前置条件** | 已有注册账号 |
| **步骤** | 1. 访问 `/auth/login`<br>2. 输入正确的邮箱和密码<br>3. 点击登录 |
| **预期结果** | 跳转到主页面 `/`，侧栏和三栏布局正常显示 |

### TC-AUTH-03：未登录时访问保护页面

| 项目 | 内容 |
|------|------|
| **前置条件** | 未登录状态 |
| **步骤** | 1. 直接访问 `/`、`/insight`、`/settings` |
| **预期结果** | 显示 loading spinner，然后自动跳转到 `/auth/login` |

### TC-AUTH-04：已登录时访问登录页

| 项目 | 内容 |
|------|------|
| **前置条件** | 已登录 |
| **步骤** | 1. 手动访问 `/auth/login` |
| **预期结果** | 自动重定向到 `/` |

### TC-AUTH-05：Demo 模式（无 Supabase）

| 项目 | 内容 |
|------|------|
| **前置条件** | `.env.local` 中无 Supabase 环境变量 |
| **步骤** | 1. 启动项目<br>2. 访问 `/` |
| **预期结果** | 不需要登录，直接显示种子数据（3 条演示 Prompt） |

| 项目 | 内容 |

| **步骤** | 1. 点击左栏右上角 "+" 按钮 |

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 填写标题：「测试 Prompt」<br>2. 填写目标：「测试目标描述」<br>3. 选择场景：Vibe Coding<br>4. 选择目标群：「前端页面生成」<br>5. 选择模型：GPT-4o<br>6. 输入 Prompt 正文<br>7. 点击「保存并补充效果」 |
| **预期结果** | ① 弹窗关闭<br>② 新 Prompt 出现在列表顶部<br>③ 自动选中新建的 Prompt<br>④ 场景自动切换到 Vibe Coding<br>⑤ 右下角 toast 提示"已保存 — 目标越具体，Insight 分析越准确"<br>⑥ 效果录入弹窗弹出 |

### TC-CAP-03：必填项校验

| **预期结果** | 「保存并补充效果」按钮灰置不可点击（需标题+目标+目标群+正文全部填写） |

### TC-CAP-04：目标群多选限制

| 项目 | 内容 |

| **预期结果** | ① 已选 3 个时，其他标签灰置不可点击<br>② 右上角显示"已选 3 / 3"<br>③ 点击已选标签可取消 |

### TC-CAP-05：目标群搜索

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 在新建弹窗的目标群区域搜索框输入「前端」 |
| **预期结果** | 仅显示包含「前端」的标签（如「前端页面生成」） |

### TC-CAP-06：新建自定义目标群

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 在新建弹窗中点击「+ 新建目标群」<br>2. 输入「自定义测试群」<br>3. 点击「添加」或回车 |
| **预期结果** | ① 新目标群出现在标签列表中<br>② 自动被选中<br>③ 后续新建 Prompt 时仍然可见 |

### TC-CAP-07：自定义模型

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 点击「+ 其他」模型按钮<br>2. 输入自定义模型名「Llama 3」 |
| **预期结果** | 「其他」按钮高亮，下方出现输入框，可输入自定义模型名称 |

### TC-CAP-08：目标描述 placeholder 随场景变化

## 五、迭代模块（Iterate）

### TC-ITER-01：编辑 Prompt

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 选中一条 Prompt<br>2. 点击「编辑」按钮<br>3. 修改标题、目标、正文、模型<br>4. 点击「保存修改」 |
| **预期结果** | ① 进入编辑模式，字段变为可编辑<br>② 保存后退出编辑模式<br>③ 修改内容立即生效<br>④ 列表中标题同步更新 |

### TC-ITER-02：取消编辑

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 进入编辑模式<br>2. 修改内容<br>3. 点击「取消」 |
| **预期结果** | 恢复为编辑前的内容，退出编辑模式 |

### TC-ITER-03：Fork 新版本

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 选中一条 Prompt（当前 v1）<br>2. 点击「Fork」<br>3. 输入版本说明（或留空）<br>4. 点击「创建 v2」 |
| **预期结果** | ① Fork 弹窗关闭<br>② 版本号变为 v2<br>③ 自动进入编辑模式（方便立即修改）<br>④ 版本历史时间轴出现 v2<br>⑤ 版本说明显示在时间轴中 |

### TC-ITER-04：Fork 回车快捷键

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 在 Fork 弹窗中输入版本说明后按 Enter |
| **预期结果** | 等同于点击「创建」按钮，直接创建新版本 |

### TC-ITER-05：版本历史时间轴

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 选中有多个版本的 Prompt<br>2. 展开版本历史（右栏底部） |
| **预期结果** | ① 时间轴显示所有版本（最新在上）<br>② 当前版本有蓝色指示器 + "当前" 标签<br>③ 点击旧版本 → 右栏预览切换到该版本效果<br>④ 出现"预览中"标签和"回到最新"按钮 |

### TC-ITER-06：星标代表作

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 在版本历史中，hover 某版本出现星标按钮<br>2. 点击星标 |
| **预期结果** | ① 该版本星标点亮（实心黄色）<br>② 其他版本的星标自动取消（全局唯一）<br>③ 再次点击可取消星标 |

### TC-ITER-07：复制 Prompt 正文

| 项目 | 内容 |
|------|------|
| **步骤** | 1. 在中栏 Prompt 正文区域点击「复制」按钮 |
| **预期结果** | Prompt 正文复制到剪贴板 |
