## 一、背景与问题定义

### 1.1 问题背景

Prompt 对 AI 输出质量的影响已被广泛认可。但个人在使用 AI 工具时面临以下痛点：

- **分散存储**：好的 Prompt 分散在聊天记录、Notion、备忘录里，用完即忘
- **效果不可追溯**：不知道哪个 Prompt 产生了好效果，更不知道为什么
- **重复劳动**：相似场景反复从零撰写，低效
- **质量无法提升**：没有系统性积累，无法从自己的历史数据中提炼规律
- **大型平台太嘈杂**：FlowGPT/PromptBase 内容庞杂，不是自己用的，找不到适合自己场景和风格的

### 1.2 核心洞察

> **目标 → Prompt → 生成效果** 三者强相关。  
> 把这三者结构化地关联起来，持续积累属于自己的高质量样本，再提炼**同一具体目标群下**的 Prompt 结构规律——才能真正让个人 Prompt 能力持续进化。

**关键粒度说明**：规律提炼不是泛泛分析"某场景下所有 Prompt"，而是聚焦**同一具体目标群**（如"生成好看的前端页面"）下的多条记录，找出高分 Prompt 共同具备的结构要素。

### 1.3 产品核心定位

> **Promptory 是你的个人专属 Prompt 知识仓库。**  
> 不是大型公共平台，不是交易市场。  
> 而是：你自己用得最顺手的、适合你的场景和 AI 工具的**私人提效仓库**，并随着积累越用越精准。

## 二、竞品深度分析

### 2.1 FlowGPT —— 失控的 UGC 娱乐化平台

**基本数据**：月访问 ~280万（2026年2月）；成立 2023年2月；Freemium + 虚拟代币（Flux）

**垂直领域**：娱乐/角色扮演占主导，学术/生产力场景被严重边缘化。月流量从 2024年顶峰 421万跌至 280万，评分仅 3.2/5。

**产品闭环**：UGC + 点赞排名 + 虚拟代币激励。典型内容平台逻辑，质量无保障。

**核心差距**：无个人积累，无效果追踪，无规律提炼。**生产力/学术/Coding 场景是被放弃的蓝海。**

---

### 2.2 PromptBase —— 高质量 Prompt 交易市场

**基本数据**：45万+用户；27万+ Prompt；成立 2022年6月；80/20分成

**垂直领域**：图像/视频生成占绝大多数收入，学术写作/知识工作/编程场景**严重空白**。

**产品闭环**：专业卖家创作 → 人工审核 → 上架（附效果示例）→ 用户购买（$1.99-$9.99）→ 平台抽 20%。  
**卖的本质**：确定性——保证用这个 Prompt 能得到预期效果，所以有人愿意付费。

**核心差距**：卖的是别人的 Prompt，不积累自己的知识；无个人规律提炼机制。

---

| 维度 | FlowGPT | PromptBase | **Promptory** |

## 四、核心功能设计

### 4.1 产品定性：纯管理工具，非执行平台

> **Promptory 不内置 AI 执行能力。**  
> 用户在 ChatGPT / Claude / Gemini 等官方平台上自行运行 Prompt，将结果回填到 Promptory 记录。  
> Promptory 只做：**记录、组织、检索、分析**。

这个边界决策带来的好处：
- 无需管理 API Key、计费、限流等复杂性
- 无需承担 AI 推理成本
- Prompt 执行结果 100% 真实（来自官方平台，非中间层）
- 产品复杂度大幅降低，MVP 更快落地

**唯一例外**：Insight 规律提炼功能需要 AI 分析 Prompt 结构（见下方专项说明）。

---

### 4.2 核心数据模型

```
Prompt 记录
├── id
├── title                    # 标题（简短描述）
├── goal                     # 目标描述（核心字段：本次想达到什么效果）
├── content                  # Prompt 正文
├── model_used               # AI 模型（预设可选 + 自定义）
├── scene                    # 一级场景（预设标签，单选）
├── goal_cluster[]           # 目标群（预设标签 + 自定义，可多选）← 规律提炼的分组依据
├── tags[]                   # 自由标签（用户自定义）
│
├── effect                   # 效果记录（分级，可延迟填写）
│   ├── level                # quick（仅评分）/ full（完整记录）
│   ├── score                # 效果评分（1-5星）
│   ├── output_content       # 真实 AI 输出内容（full 级别）
│   └── notes                # 补充备注
│
├── versions[]               # 版本历史
│   └── {ver, content, effect, updated_at}
│
├── is_public                # 是否公开（默认私有，MVP 阶段均私有）
└── created_at / updated_at
```

**各栏职责**

| 栏 | 宽度（参考） | 核心内容 | 交互 |
|----|------------|---------|------|

```

```

**Insight 入口**：左栏每个场景 Tab 右侧有"Insight ✦"按钮，达到触发阈值后高亮；点击进入该场景下所有目标群的提炼报告页。

### 4.4 冷启动：导入向导（Import Wizard）

> 解决"第一次打开是空白"的体感问题。用户约有 10+ 条历史 Prompt 可整理。

**向导流程**（首次登录后自动触发，也可手动进入）

```
Step 1  欢迎页
        "把你积累的 Prompt 搬进来，Promptory 帮你整理成知识库"
        [开始导入]  [跳过，直接使用]

Step 2  选择导入方式
        ① 逐条手动添加（引导式表单）
        ② 批量粘贴文本（结构化解析）← 重点
        ③ 从 Notion / CSV 导入（后续版本）

Step 3  批量粘贴模式
        提供标准格式模板，用户按格式粘贴多条：
        ---
        标题：生成响应式登录页
        目标：用 React+Tailwind 生成现代风 SaaS 登录页
        Prompt：你是一位...（正文）
        模型：GPT-4o
        评分：4
        ---
        系统解析后展示预览，用户逐条确认/微调

Step 4  完成
        "已导入 12 条 Prompt，分布在 3 个场景"
        [进入 Promptory]
```

**设计原则**：向导不强制完整填写效果记录，目的是快速把"已有的好 Prompt"先录进来，效果可以后续补。

### 4.5 功能模块详细设计

#### 模块一：录入（Capture）

**新建 Prompt 弹窗/侧滑面板**（不跳页，在主界面内完成）

```
Step 1  标题（必填）
        目标描述（必填）— 动态 placeholder 随场景变化：
          Vibe Coding → "例：用 React + Tailwind 生成现代风 SaaS 登录页"
          学术写作   → "例：润色这段研究背景，符合 Nature 期刊风格"
          PM         → "例：基于以下需求生成完整的用户故事列表"

Step 2  Prompt 正文（必填，大文本域）

Step 3  元信息选择（快速选择器）
        · AI 模型    [GPT-4o ▾]（预设列表 + 自定义）
        · 场景       [Vibe Coding ▾]
        · 目标群     [多选标签面板，预设+自定义]

Step 4  [保存] → 效果记录可选填：
        · 快速档：点评分（1-5星），30秒
        · 完整档：粘贴真实 AI 输出 + 评分 + 备注
        · [跳过，稍后补充] → 右栏显示"待补充效果"状态
```

**设计原则**：效果记录不强制，可先保存 Prompt，任意时间回来在右栏补充。

---

#### 模块二：查询（Search）

- **全文检索**：搜索匹配目标/标题/标签/Prompt 内容
- **筛选维度**：场景 / AI 模型 / 目标群 / 效果评分范围
- **排序选项**：最近使用 / 评分最高 / 最多版本（迭代最多 = 打磨最精）
- **同目标推荐**：查看某条 Prompt 时，侧边显示"同目标群的其他记录（N条）"

---

#### 模块三：迭代（Iterate）

- 基于任意 Prompt 一键 Fork 为新版本
- Fork 时弹出"版本说明"输入框（选填，可跳过）：`例：细化了视觉风格描述，增加了交互动效要求`
- **版本时间轴**（右栏下半区）：

```
右栏结构
┌─────────────────────────────────┐
│  ☆ 置顶效果（用户点亮的最佳效果） │  ← 有点亮时显示，无则隐藏此区块
│  ★★★★★  GPT-4o  v2            │
│  [AI 输出内容预览...]            │
├─────────────────────────────────┤
│  当前展示：最新版本 v3 的效果     │  ← 默认展示最新版本效果
│  ★★★★☆  Claude 3.7  v3        │
│  [AI 输出内容...]               │
│  备注：...                      │
│  [补充/编辑效果]                │
├─────────────────────────────────┤
│  版本历史                        │
│  ● v3  ★★★★☆  04-17  当前     │
│  ○ v2  ★★★★★  04-12  ☆点亮   │  ← 已点亮标识
│  ○ v1  ★★☆☆☆  04-05          │
│  （点击任意版本 → 切换上方预览）  │
└─────────────────────────────────┘
```

**点亮（☆ → ★）机制**：
- 在版本历史中，对任意版本的效果点击☆图标 → 该效果置顶显示在右栏顶部
- 每条 Prompt 只能有一个置顶效果（点亮新的 → 旧的自动取消）
- 置顶效果同时作为该 Prompt 在左栏列表卡片中的"代表效果"预览

**版本对比**：点击中栏"对比版本"按钮 → 弹出左右 Diff 视图，高亮 Prompt 文字变更处，并排显示两版本评分

---

#### 模块四：规律提炼（Insight）—— 核心差异化，v0.5 上线

**触发条件**：同一目标群标签下，有效果评分的记录 ≥ 5 条时自动触发分析

**分析逻辑**：
1. 取该目标群评分 ≥ 4 的 Prompt（高分组）
2. 取评分 ≤ 2 的 Prompt（低分组，若有）
3. AI 对比分析两组，找出高分组共有、低分组缺失的结构要素
4. 生成结构化最佳模板

**输出示例（以 Vibe Coding / 前端页面生成 为例）**：

```
📊 目标群：前端页面生成
   已积累 8 条 · 高分（≥4星）6 条 · 低分（≤2星）1 条

Promptory 分析了你的高分 Prompt，发现它们都具备：

✅ 技术栈声明        "使用 React + Tailwind，不用其他 UI 库"
✅ 视觉风格约束      "现代简洁风格，参考 Linear/Vercel 设计语言"
✅ 组件结构定义      "页面包含：导航栏 + Hero区 + 功能模块 + CTA"
✅ 响应式要求        "需适配移动端，断点清晰"
✅ 交互细节说明      "hover 状态、过渡动画要求"

❌ 低分 Prompt 普遍缺失：视觉风格约束 + 交互细节说明

── 你的最佳模板（点击使用） ──────────────────────
你是一位精通现代前端设计的工程师。
使用 [技术栈] 实现以下页面，风格参考 [设计参考]。

页面结构：[组件列表]
视觉要求：[风格描述，如：简洁现代，主色 #xxx]
响应式：[断点要求]
交互：[动效说明]

请直接输出完整可运行代码。
─────────────────────────────────────────────
[用此模板创建新 Prompt]   [查看支撑数据]
```

**更新机制**：每新增 3 条有效果记录后，自动重新分析，模板持续进化

**入口**：
- 主动：每个目标群页面有"Insight"按钮（达到阈值后高亮激活）
- 被动：达到阈值时，首页顶部通知栏提示"你的「前端页面生成」目标群已可生成最佳模板"

### 4.4 整体工作流

```
遇到需求
  ↓
搜索 Promptory：有可复用的 Prompt 或最佳模板吗？
  ├── 有模板 → 基于模板填变量 → 执行
  ├── 有相似记录 → Fork 修改 → 执行
  └── 无 → 新建（填目标+Prompt+模型+目标群标签）→ 执行
                        ↓
             回来补效果（快速档 or 完整档，可延迟）
                        ↓
             同目标群积累 ≥ 5 条（有评分）
                        ↓
             Insight 触发 → 生成最佳模板
                        ↓
             下次用模板 → 效果更稳定 → 继续积累
                        ↓
                  Prompt 能力螺旋上升
```

## 五、技术架构

### 5.1 产品定性：纯管理工具，非执行平台

> **Promptory 不内置 AI 执行能力。**  
> 用户在 ChatGPT / Claude / Gemini 等官方平台自行运行 Prompt，将结果回填到 Promptory 记录。  
> Promptory 只做：**记录、组织、检索、分析**。

**工作流与 AI 平台的关系**：
```
Promptory（写 Prompt + 管理）
    ↕ 用户手动复制粘贴
官方 AI 平台（执行 Prompt，看结果）
    ↕ 用户手动复制粘贴
Promptory（回填效果，积累数据）
```

这个边界决策带来的好处：
- 无需管理 API Key、计费、限流
- 执行结果来自官方平台，真实可信
- 产品复杂度大幅降低，MVP 更快落地

---

### 5.2 Insight 的 AI 分析方案：用户自填 API Key（方案B）

**定案**：Insight 做结构语义分析，用户在设置中填入自己的 AI API Key，Promptory 后端不存储、不中转 Prompt 内容，直接由前端或用户服务器调用 AI API 分析已积累数据。

**支持的 API 提供商（按优先级）**

| 提供商 | 免费额度 | 适合场景 | 优先级 |
|--------|---------|---------|--------|
| **Kimi（月之暗面）** | 有免费额度，性价比高 | 中文 Prompt 分析尤佳 | **P0，首选** |
| **Google Gemini** | 免费 tier 较慷慨 | 通用分析 | P0 |
| **DeepSeek** | 价格极低（几乎免费） | 结构分析 | P1 |
| **OpenAI GPT** | 无免费，但用量极低（$0.01/次） | 兜底 | P2 |
| **Claude** | 无免费 | 兜底 | P2 |

**用户配置流程**（Settings 页）：
```
AI 分析设置
├── 选择提供商：[Kimi ▾]
├── 填入 API Key：[sk-xxxx____________]  [测试连接]
└── 保存
```

**隐私保障**：
- API Key 加密存储在 Supabase，不明文传输
- Insight 分析时，只将 Prompt 正文 + 评分发送给 AI API，不发送用户个人信息
- 每次分析预估 token 消耗，在触发前提示用户："本次分析预计消耗约 2000 tokens"

**单次分析成本估算**：分析 5-10 条 Prompt，约 2000-4000 tokens，Kimi / DeepSeek 成本 < ¥0.01，几乎无感。

### 5.3 技术选型

| 层 | 方案 | 备注 |
|----|------|------|
| 前端 | Next.js (React) | PC Web 优先，三栏布局 |
| 后端 | Node.js + Prisma | 全栈简洁 |
| 数据库 | **Supabase (PostgreSQL)** | 托管、含 pgvector、内置 Auth |
| 语义检索 | pgvector | 目标描述相似度检索 |
| Insight 分析 | 规则统计（MVP）→ 用户 API Key（v0.5） | 见上方方案说明 |
| 数据导出 | JSON / CSV 导出接口 | 用户随时可带走全部数据 |
| 部署 | Vercel | 快速上线 |

### 5.4 数据安全与隐私

- **存储**：Supabase 托管，数据库行级安全（RLS），每个用户只能访问自己的数据
- **传输**：全程 HTTPS
- **导出**：提供一键全量导出（JSON/CSV），用户随时可完整带走自己的数据，不绑定平台
- **Insight 分析**：MVP 阶段规则统计，Prompt 数据不离开 Supabase；方案B下用户自己的 Key 直连 AI 平台，Promptory 后端不中转 Prompt 内容
- **敏感数据提示**：录入页注明"你的数据仅供你个人访问，默认不对外可见"
